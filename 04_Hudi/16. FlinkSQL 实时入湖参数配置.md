# 基础表类型参数
## 表类型
```sql
-- 指定 Hudi 表类型
'table.type' = 'MERGE_ON_READ'  -- 或 'COPY_ON_WRITE'
```

生产建议：

+ `MERGE_ON_READ` (MOR)：高频写入 + 更新频繁的场景（如实时数仓 DWD 层）
+ `COPY_ON_WRITE` (COW)：低频写入或 append-only的场景（如日志存储）

## 主键与预合并
```sql
-- 主键字段
'hoodie.datasource.write.recordkey.field' = 'order_id'

-- 预合并字段（用于处理同一主键的多条记录）
'hoodie.datasource.write.precombine.field' = 'ts'
```

生产建议：

+ 必须指定主键，否则无法去重
+ 预合并字段通常选择时间戳字段，确保最新数据被保留

# 写入性能与并行度参数
## 写入并行度
```sql
-- 写入任务的并行度
'write.tasks' = '4'

-- 分区排序（提升写入效率）
'hoodie.bulkinsert.sort.mode' = 'GLOBAL_SORT'  -- 或 'PARTITION_SORT'
```

生产建议：

+ `write.tasks`：建议设置为 Kafka 分区数的 1-2 倍，或根据数据量设置（单 task 处理 50-100MB/s 为宜）
+ 排序模式：
    - `GLOBAL_SORT`：全局排序，小文件最少但 shuffle 开销大（适合 bulk insert）
    - `PARTITION_SORT`：分区内排序，平衡性能与文件大小（推荐实时写入）

## 写入方式
```sql
-- 写入操作类型
'write.operation' = 'upsert'  -- 或 'insert', 'bulk_insert'
```

生产建议：

+ 实时流：`upsert`（默认）
+ 初次批量导数据：`bulk_insert`（关闭去重，写入更快）
+ append-only：`insert`

# 文件大小控制参数
## 目标文件大小
```sql
-- 目标文件大小（字节）
'hoodie.parquet.max.file.size' = '536870912'  -- 512MB
'hoodie.parquet.small.file.limit' = '104857600'  -- 100MB 小文件阈值
```

生产建议：

+ 大表：512MB - 1GB
+ 小表/高更新率表：256MB
    - `small.file.limit` 建议设置为目标文件的 1/5 ~ 1/4，低于此值会尝试合并

## 文件滚动控制
```sql
-- 基于记录数滚动文件
'hoodie.parquet.max.records.per.file' = '5000000'  -- 500万条

-- 基于时间滚动（毫秒）
'hoodie.parquet.file.rolling.time.seconds' = '3600'  -- 1小时
```

生产建议：

+ 同时设置记录数和时间双重保障
+ 例如：500万条或 1小时，哪个先到就滚动

# 索引类型参数（关键！）
## 索引类型
```sql
-- 索引类型
'hoodie.index.type' = 'BUCKET'  -- 或 'GLOBAL_BLOOM', 'FLINK_STATE'
'hoodie.bucket.index.num.buckets' = '128'  -- BUCKET 索引的分桶数
```

生产选型对比：

| 索引类型 | 适用场景 | 优点 | 缺点 |
| --- | --- | --- | --- |
| BUCKET | 大表高频更新（推荐） | 无状态，去重快 | 分桶数需预先设定 |
| GLOBAL_BLOOM | 中等规模表 | 节省状态后端空间 | 有误判风险，需调优 |
| FLINK_STATE | 小表 | 精确去重 | 状态后端压力大 |


生产建议：优先选择 `BUCKET` 索引，特别是 0.12+ 版本

+ 分桶数设置：数据量 / 每个桶 2-5GB（如 1TB 数据，设 256-512 个桶）

# 一致性保障参数
## Exactly-Once 语义
```sql
-- 开启最终一致性写入
'write.insert.drop.duplicates' = 'false'  -- 是否丢弃重复

-- Flink 端 Exactly-Once
'execution.checkpointing.mode' = 'EXACTLY_ONCE'
'sink.semantic' = 'exactly-once'  -- Hudi 0.13+ 支持
```

生产建议：

+ 必须开启 Checkpoint，且设置合理间隔（如 1-5 分钟）
+ Hudi 0.13 以上可设置 `sink.semantic = 'exactly-once'`，否则是 at-least-once

## 并发写入控制
```sql
-- 是否允许并发写入同一张表
'hoodie.write.concurrency.mode' = 'SINGLE_WRITER'  -- 或 'OPTIMISTIC_CONCURRENCY_CONTROL'
```

生产建议：

+ 绝大多数场景：`SINGLE_WRITER`（单作业写入）
+ 多作业写入同一表：需开启 OCC 并配置锁提供者（ZK/Hive）

# Hive 同步参数
## 元数据同步
```sql
-- 同步 Hive Metastore
'hive_sync.enable' = 'true'
'hive_sync.metastore.uris' = 'thrift://hive-metastore:9083'
'hive_sync.db' = 'default'
'hive_sync.table' = 'hudi_table'
'hive_sync.mode' = 'hms'  -- 或 jdbc
```

生产建议：

+ 必须开启，否则查询引擎无法感知新分区
+ 使用 `hms` 模式（直接连接 HMS），避免 JDBC 方式

## 分区同步
```sql
-- 分区字段
'hoodie.datasource.write.partitionpath.field' = 'dt'

-- 分区自动同步
'hive_sync.partition.fields' = 'dt'
'hive_sync.partition.extractor' = 'org.apache.hudi.hive.MultiPartKeysValueExtractor'
```

生产建议：

+ 分区字段选择日期/小时，便于后续查询裁剪
+ 分区提取器使用 `MultiPartKeysValueExtractor`（支持多级分区）

# 内存与性能调优参数
## 写入端内存
```sql
-- Flink 算子内存
'write.task.max.size' = '1024'  -- MB，默认 1024

-- Hudi 内部内存
'hoodie.memory.merge.max.size' = '2147483648'  -- 2GB
'hoodie.memory.compaction.max.size' = '2147483648'  -- 2GB
```

生产建议：

+ 如果数据体量大，适当调高 `write.task.max.size` 到 2048-4096
+ 每个 TM 的内存分配要考虑 Hudi 合并操作的开销

## 写入重试机制
```sql
-- 写入失败重试
'hoodie.write.commit.retry.count' = '5'
'hoodie.write.commit.retry.interval.ms' = '2000'
```

生产建议：开启重试，避免瞬态故障导致作业失败

# <font style="color:rgb(15, 17, 21);">Flink 作业必须配置的 Checkpoint 参数</font>
Flink SQL 实时写入作业本身仍然必须配置 Checkpoint。这是因为 Hudi 利用了 Flink 的 Checkpoint 机制来实现两阶段提交协议，保证数据的Exactly-Once 写入。没有 Checkpoint，Hudi 写入器无法确认事务并提交数据，最终会导致数据无法可见或作业卡住

## <font style="color:rgb(15, 17, 21);"></font>**<font style="color:rgb(15, 17, 21);">Checkpoint 基础配置</font>**
```plsql
-- 必须开启checkpoint！Hudi依赖它提交事务
set execution.checkpointing.interval = '300000';        -- 5分钟，建议分钟级[citation:8]

-- checkpoint模式
set execution.checkpointing.mode = 'EXACTLY_ONCE';      -- 保证精确一次语义

-- checkpoint超时
set execution.checkpointing.timeout = '600000';          -- 10分钟

-- checkpoint保留策略
set execution.checkpointing.externalized-checkpoint-retention = 'RETAIN_ON_CANCELLATION';  -- 取消作业时保留checkpoint[citation:10]
```

## <font style="color:rgb(15, 17, 21);"></font>**<font style="color:rgb(15, 17, 21);">状态后端与存储路径</font>**
```plsql
-- checkpoint存储目录
set state.checkpoints.dir = 'hdfs:///flink/checkpoints/order_writer';

-- 状态后端（推荐RocksDB）
set state.backend = 'rocksdb';
set state.backend.incremental = 'true';                  -- 开启增量checkpoint
```

## **<font style="color:rgb(15, 17, 21);">重启策略与容错</font>**
```plsql
-- 重启策略
set restart-strategy = 'fixed-delay';
set restart-strategy.fixed-delay.attempts = '10';
set restart-strategy.fixed-delay.delay = '30s';
```

# 生产环境推荐配置模板
场景：实时数仓 DWD 层（MOR 表，高频更新）

先配置 Flink，参考上一节，然后配置hudi sink

```sql
CREATE TABLE dwd_order (
  order_id STRING,
  user_id STRING,
  amount DOUBLE,
  ts TIMESTAMP(3),
  dt STRING,
  PRIMARY KEY(order_id) NOT ENFORCED
) WITH (
  'connector' = 'hudi',
  'path' = 'hdfs:///user/hudi/dwd_order',
  
  -- 表类型
  'table.type' = 'MERGE_ON_READ',
  'hoodie.datasource.write.recordkey.field' = 'order_id',
  'hoodie.datasource.write.precombine.field' = 'ts',
  
  -- 索引（关键！）
  'hoodie.index.type' = 'BUCKET',
  'hoodie.bucket.index.num.buckets' = '256',
  
  -- 写入并行度
  'write.tasks' = '8',
  'write.operation' = 'upsert',
  
  -- 文件大小控制
  'hoodie.parquet.max.file.size' = '536870912',  -- 512MB
  'hoodie.parquet.small.file.limit' = '134217728',  -- 128MB
  
  -- Compaction（异步高频）
  'compaction.async.enabled' = 'true',
  'compaction.delta_commits' = '5',
  'compaction.tasks' = '4',
  
  -- Clean（低频清理）
  'clean.async.enabled' = 'true',
  'clean.delta_commits' = '15',
  'hoodie.keep.max.commits' = '30',
  'hoodie.cleaner.retainer.hours' = '72',
  
  -- 分区
  'hoodie.datasource.write.partitionpath.field' = 'dt',
  
  -- Hive 同步
  'hive_sync.enable' = 'true',
  'hive_sync.metastore.uris' = 'thrift://hive-metastore:9083',
  'hive_sync.db' = 'dwd',
  'hive_sync.table' = 'dwd_order',
  'hive_sync.partition.fields' = 'dt',
  'hive_sync.partition.extractor' = 'org.apache.hudi.hive.MultiPartKeysValueExtractor'
)
```

# 总结：必设参数清单
| 类别 | 参数 | 生产推荐值 |
| --- | --- | --- |
| 表类型 | `table.type` | `MERGE_ON_READ`（更新频繁）/ `COPY_ON_WRITE`（只读） |
| 索引 | `hoodie.index.type` | `BUCKET`（0.12+ 大表推荐） |
| 主键 | `hoodie.datasource.write.recordkey.field` | 业务主键 |
| 预合并 | `hoodie.datasource.write.precombine.field` | 时间戳字段 |
| 文件大小 | `hoodie.parquet.max.file.size` | 512MB - 1GB |
| 写入并行度 | `write.tasks` | Kafka 分区数 × 1-2 |
| Compaction | `compaction.delta_commits` | 3-10（高频表取小值） |
| Clean | `clean.delta_commits` | 10-15 |
| Hive 同步 | `hive_sync.enable` | `true` |


这些参数组合配置得当，才能保证 Hudi 入湖作业在生产环境稳定、高效运行。
