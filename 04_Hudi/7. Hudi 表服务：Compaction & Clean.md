# 表服务简介
数据湖表通常需要运行一些公共服务来维护表的性能和管理存储，这些在 Hudi 中被称为表服务（Table Services），表服务是幂等的（可重复执行，结果一致）

+ `CLEANS`：清理旧版本数据（类似垃圾回收）
+ `COMPACTION`：合并增量日志到基础文件



表服务可以同步运行，也可异步执行

+ 同步：Hudi 支持 job 内`inline compation`和`clean`，可以及时合并小文件并清理，从而避免了小文件问题。
+ 异步：也可以通过参数，关闭`inline compaction`，Hudi 在 Spark/Flink 中都提供了`offline compaction`和`clean`

# Compaction 操作
## 小文件问题
小文件问题产生的原因：以 Flink 流式写入为例

+ 每个 checkpoint 生成一批文件
+ 每次写入数据量可能不大
+ 导致产生大量小文件（几MB甚至更小）

小文件导致的问题

+ 查询性能差（需要打开很多文件）
+ 元数据管理开销大
+ 存储效率低

## Hudi 解决方案：Compaction
1. 内联 Compaction（Inline Compaction）
    - 工作原理：在写入作业中同步执行压缩

```sql
-- 启用内联压缩
'compaction.async.enabled' = 'false',     -- 关闭异步压缩
'compaction.schedule.enabled' = 'true',   -- 启用压缩调度
'compaction.delta_commits' = '5',         -- 每5次提交触发一次压缩
'compaction.max_memory' = '1024',         -- 压缩内存大小（MB）
```

    - 优点：及时合并小文件，查询性能始终最优
    - 缺点：增加写入延迟，可能影响写入吞吐量
2. 异步 Compaction（Async Compaction）
    - 工作原理：由独立作业执行压缩

```sql
-- 启用异步压缩
'compaction.async.enabled' = 'true',      -- 启用异步压缩
'compaction.schedule.enabled' = 'true',   -- 写入作业生成压缩计划
'compaction.tasks' = '3',                 -- 压缩任务并发度
'compaction.trigger.strategy' = 'num_commits',  -- 按提交次数触发
'compaction.delta_commits' = '10'         -- 每10次提交触发
```

    - 优点：写入作业不受压缩影响，可以单独控制压缩资源，支持离线压缩（offline compaction）
    - 缺点：查询可能看到未压缩的数据，需要额外的作业管理

# Clean 操作
Hudi 保留数据的历史版本（用于时间旅行）  
但需要定期清理：

+ 过期的快照
+ 失败的提交残留
+ 临时文件



1. 内联 Clean 配置

```sql
'clean.async.enabled' = 'false',          -- 内联清理
'cleaner.policy' = 'KEEP_LATEST_COMMITS', -- 保留策略
'cleaner.commits.retained' = '10',        -- 保留最近10次提交
'cleaner.fileversions.retained' = '3'     -- 保留3个文件版本
```

2. 异步 Clean 配置

```sql
'clean.async.enabled' = 'true',           -- 异步清理
'clean.trigger.strategy' = 'num_commits', -- 触发策略
'clean.delta_commits' = '5'               -- 每5次提交触发清理
```

# Flink 中配置 Compaction & Clean
生产环境推荐使用异步

```sql
-- 写入作业只负责写入，不执行压缩
CREATE TABLE hudi_table (
  ...
) WITH (
  'connector' = 'hudi',
  'table.type' = 'MERGE_ON_READ',
  
  -- 异步压缩配置
  'compaction.async.enabled' = 'true',
  'compaction.schedule.enabled' = 'true',
  
  -- 异步清理配置
  'clean.async.enabled' = 'true',
  
  -- 写入作业只生成计划
  'write.operation' = 'upsert'
);

-- 需要另外启动压缩作业
-- 可以通过 Flink CLI 或脚本调度
```
