核心思想：所有写入作业只负责写入（log files），Compaction 和 Clean 由独立的专用作业统一调度执行。

# 推荐架构
| 作业类型 | 功能 | StreamPark 配置要点 |
| --- | --- | --- |
| 写入作业（多个） | 实时写入 Kafka/MySQL CDC → Hudi MOR 表 | 关闭 compaction & clean |
| 维护作业（1个） | 定期执行 Compaction + Clean | 仅执行维护，不写数据 |


# 具体配置方案
## 所有写入作业（FlinkSQL DDL 中配置）
```plsql
CREATE TABLE hudi_table (
  id STRING,
  name STRING,
  ts TIMESTAMP(3),
  PRIMARY KEY (id) NOT ENFORCED
) WITH (
  'connector' = 'hudi',
  'path' = 'hdfs://.../hudi_table',
  'table.type' = 'MERGE_ON_READ',

  -- ⛔ 禁用写入作业的 compaction 和 clean
  'compaction.async.enabled' = 'false',
  'compaction.schedule.enabled' = 'true',    -- 只做调度
  'clean.async.enabled' = 'false',

  -- 可选：控制小文件合并（但不触发 compaction）
  'hoodie.parquet.max.file.size' = '134217728',      -- 128MB
  'hoodie.parquet.small.file.limit' = '104857600',   -- 100MB
  
  -- 在 CREATE TABLE 时定义（所有写入作业需一致）
  'hoodie.cleaner.policy' = 'KEEP_LATEST_COMMITS',     -- 默认
  'hoodie.cleaner.commits.retained' = '24',            -- 保留最近 24 次 commits（约24小时）
  'hoodie.clean.async' = 'false'                       -- 由维护作业显式触发

  'write.operation' = 'upsert',
  'write.batch.size' = '128'
);
```

## 单独的表服务作业（Flink SQL）
创建一个专用的 Compaction & Clean 维护作业，FlinkSQL

```plsql
-- 执行一次 compaction（可配合调度器周期运行）
EXECUTE COMPACT TABLE hudi_table;

-- 执行一次 clean（通常 compaction 后自动触发，也可显式调用）
EXECUTE CLEAN TABLE hudi_table;
```

在 StreamPark 中，可将此 SQL 作为 批处理作业，通过 定时调度（如每小时/每天）运行。



调度策略

1. 每天至少一次
2. 建议 2 ~ 4 小时执行一次

维护作业必须串行执行（不能并发），可在 StreamPark 中设置 作业互斥锁 或使用 单实例调度。

## 单独表服务作业（Shell）
```shell
./bin/flink run \
-c org.apache.hudi.sink.compact.HoodieFlinkCompactor \
lib/hudi-flink1.13-bundle-0.12.0.jar \

```

# Clean 任务
写入作业异步做 Clean，彻底分离 写入和 Clean

1. 彻底分离写入任务和 Clean：这是最彻底的资源隔离方式，Clean任务独立于写入作业运行，互不影响。
    1. 方法A：使用 Spark 作业运行 Hudi 官方 Cleaner（当前最稳定）
        1. Flink引擎侧目前没有像Compaction那样提供独立的官方Clean工具。生产上最通用的做法是提交一个短暂的Spark作业来执行清理。这是Apache Hudi官方提供的工具，非常稳定
    2. 方法B：使用 Spark SQL 手动触发（适合脚本调度）
    3. 方案C：如果只有Flink集群：可以写一个简单的Flink批处理作业，调用Hudi的Clean API（Hudi 0.13+支持），但成熟度不如Spark版本。
2. 在写入作业中开启异步 Clean（简单但共享资源），让同一个Flink SQL写入作业在后台线程执行清理
