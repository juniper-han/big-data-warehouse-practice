`LATERAL VIEW` 是 HiveSQL 中处理复杂数据类型（如 `ARRAY`、`MAP`、`STRUCT`）进行“行转列”展开 的核心语法。它通常与表生成函数（Table-Generating Functions, TGF） 搭配使用，将一行中的复杂字段“炸开”成多行，同时保留原表的其他字段。

# `LATERAL VIEW` 简介
## 引入必要性
Hive 支持复杂数据类型：

+ `ARRAY<type>`：数组
+ `MAP<key_type, value_type>`：键值对
+ `STRUCT<col1:type1, col2:type2>`：结构体

但这些类型无法直接在 `SELECT` 中像普通列一样使用。

例如：一个用户有多个手机号（存为 `ARRAY<string>`）

+ 你想把每个手机号单独成一行，同时保留用户 ID。

这就需要 “行转列”（Explode），而 LATERAL VIEW 就是实现这一功能的标准方式。

## 基本语法
```plsql
SELECT 
    original_columns,
    alias.column_from_udtf
FROM table_name
LATERAL VIEW udtf(expression) table_alias AS column_alias1, column_alias2, ...;
```

参数

+ `udtf(expression)`：调用一个表生成函数（如`EXPLODE()`）
+ `table_alias`：为 UDTF 生成的虚拟表起别名（可选，但推荐）
+ `column_alias`...	：为 UDTF 输出的每一列起别名（必须）



`LATERAL VIEW` 本质是：对原表的每一行，应用一个函数生成一个临时表，然后与原表做 `INNER JOIN`

# 搭配函数（UDTF）
Hive 提供了几个内置的表生成函数（UDTF），必须与 `LATERAL VIEW` 联合使用

UDTF 只能在 `LATERAL VIEW` 或 `SELECT` 中（作为表源）使用，不能在 `WHERE`中使用

## `EXPLODE(array) / EXPLODE(map)`
作用：将 ARRAY 或 MAP “炸开”成多行。

输出列数：

+ `EXPLODE(ARRAY) AS col`→ 1 列（元素值）
+ `EXPLODE(MAP) AS key, value`→ 2 列`（key, value）`

注意事项：

+ 必须指定`AS`后的列名
+ `EXPLODE` 会显著增加数据行数，可能导致 shuffle 压力大。
+ 尽量在 LATERAL VIEW 前加过滤条件，减少输入数据量。



### 示例 1：ARRAY 炸开：
```plsql
-- 原表 users: id (INT), phones (ARRAY<STRING>)
SELECT id, phone
FROM users
LATERAL VIEW explode(phones) t AS phone;
```

输入：

| id | phones |
| --- | --- |
| 1 | ["138", "139"] |


输出：

| id | phone |
| --- | --- |
| 1 | 138 |
| 1 | 139 |


### 示例 2：MAP 炸开
```plsql
-- tags: MAP<STRING, INT>，如 {"vip":1, "new":2}
SELECT id, tag_key, tag_value
FROM users
LATERAL VIEW explode(tags) t AS tag_key, tag_value;
```

## `POSEXPLODE(array) / POSEXPLODE(map)`
作用：类似 `EXPLODE`，但额外输出元素的位置索引（从 0 开始）

输出列：

+ `POSEXPLODE(ARRAY) `→ `pos`, `val`
+ `POSEXPLODE(MAP)` → `pos`, `key`, `value`



```plsql
SELECT id, pos, phone
FROM users
LATERAL VIEW posexplode(phones) t AS pos, phone;
```

输入：

| id | phones |
| --- | --- |
| 1 | ["138", "139"] |


输出

| id | pos | phone |
| --- | --- | --- |
| 1 | 0 | 138 |
| 1 | 1 | 139 |


## `JSON_TUPLE(json_str, field1, field2, ...)`
作用：从 JSON 字符串中提取多个字段（比 `get_json_object` 更高效）

注意：只能用于 JSON 对象（{}），不能处理数组

```plsql
-- log: '{"name":"Alice","age":25}'
SELECT id, name, age
FROM logs
LATERAL VIEW json_tuple(log, 'name', 'age') t AS name, age;
```

⚠️ `json_tuple` 不支持嵌套 JSON（如 a.b.c），此时仍需 `get_json_object`。

# 高级用法
## 多个 `LATERAL VIEW`（嵌套展开）
```plsql
-- 先炸开 map，再炸开 map 中的 array 值
SELECT id, tag, phone
FROM users
LATERAL VIEW explode(user_tags) t1 AS tag, phones_list
LATERAL VIEW explode(phones_list) t2 AS phone;
```

## 保留 `NULL` 行：`LATERAL VIEW OUTER`
默认`LATERAL VIEW`是`INNER JOIN`行为：如果 UDTF 输入为`NULL`或空数组，该行会被丢弃。

若想保留原表行（即使炸开后无数据），用 `OUTER`：

```plsql
SELECT id, phone
FROM users
LATERAL VIEW OUTER explode(phones) t AS phone;
```

+ 若 `phones = NULL` 或 `[]`，则输出 `id`, `NULL`
+ 默认（无 `OUTER`）则完全不输出该行
