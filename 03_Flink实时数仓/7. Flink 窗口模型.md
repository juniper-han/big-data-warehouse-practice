# 窗口概念
## 窗口简介
+ 针对无界数据流，我们无法等待整个数据流终止后才进行统计。通常情况下，我们只需要对某个时间范围，或者数量范围内的数据进行统计分析，如：
    - 每隔五分钟统计一次过去一小时内所有商品的点击量
    - 或者每发生 1000 次点击后，都去统计一下每个商品点击率的占比
+ 在 Flink 中，使用窗口（Window）来实现这类功能
+ Flink 支持在 Stream 上通过 Key 区分多个窗口



Window：是在无限的流上定义了一个有限的元素集合。这个集合可以是基于时间的，元素个数的，时间和个数结合的，会话间隙的，或者是自定义的。

## 窗口类型
按照统计维度的不同，Flink 中的窗口可以分为

+ 时间窗口：Time Windows
    - 滚动窗口：Tumbling Windows，不重叠
    - 滑动窗口：Sliding Windows，重叠
    - 会话窗口：Session Windows
    - 全局窗口：Global Windows
+ 计数窗口：Count Windows
    - 滚动窗口
    - 滑动窗口

## 窗口属性
<font style="color:rgb(64, 72, 91);">Flink 支持窗口的两个重要属性</font>

+ <font style="color:rgb(64, 72, 91);">size：窗口长度</font>
+ <font style="color:rgb(64, 72, 91);">interval：窗口之间的间隔</font>

size 和 interval 组合

+ <font style="color:rgb(64, 72, 91);">如果 size = interval：滚动窗口，tumbling-window，无重叠数据</font>
+ <font style="color:rgb(64, 72, 91);">如果 size > interval：滑动窗口，sliding-window，有重叠数据</font>
+ <font style="color:rgb(64, 72, 91);">如果 size < interval，这种窗口将会丢失数据。比如每 5 秒钟，统计过去 3 秒的通过路口汽车的数据，将会漏掉 2 秒钟的数据</font>

## <font style="color:rgb(64, 72, 91);">四种基本窗口的实现</font>
<font style="color:rgb(64, 72, 91);">通过组合可以得出四种基本窗口：</font>

+ <font style="color:rgb(64, 72, 91);">无重叠数据的时间窗口：滚动窗口</font>
    - <font style="color:rgb(64, 72, 91);">time-tumbling-window</font>
    - <font style="color:rgb(64, 72, 91);">设置方式：</font>`<font style="color:rgb(64, 72, 91);">timeWindow(Time.seconds(5))</font>`
+ <font style="color:rgb(64, 72, 91);">有重叠数据的时间窗口：滑动窗口</font>
    - <font style="color:rgb(64, 72, 91);">time-sliding-window</font>
    - <font style="color:rgb(64, 72, 91);">设置方式：</font>`<font style="color:rgb(64, 72, 91);">timeWindow(Time.seconds(5), Time.seconds(3))</font>`
+ <font style="color:rgb(64, 72, 91);">无重叠数据的数量窗口</font>
    - <font style="color:rgb(64, 72, 91);">count-tumbing-window</font>
    - <font style="color:rgb(64, 72, 91);">设置方式：</font>`<font style="color:rgb(64, 72, 91);">countWindow(5)</font>`
+ <font style="color:rgb(64, 72, 91);">有重叠数据的数量窗口</font>
    - <font style="color:rgb(64, 72, 91);">count-sliding-window</font>
    - <font style="color:rgb(64, 72, 91);">设置方式：</font>`<font style="color:rgb(64, 72, 91);">countWindow(5,3)</font>`

# Time Windows
Time Windows 用于以时间为维度来进行数据聚合，具体分为以下四类：

## Tumbling Windows
滚动窗口：Tumbling Windows：

+ 是指彼此之间没有重叠的窗口
+ 每一个事件只能属于一个窗口

例如：每隔1小时统计过去1小时内的商品点击量，那么 1 天就只能分为 24 个窗口，每个窗口彼此之间是不存在重叠的，具体如下：

<img src="https://cdn.nlark.com/yuque/0/2025/png/54426354/1758288929963-a5614dc0-ec7b-4396-99eb-746e24f4c135.png" width="834" title="" crop="0,0,1,1" id="uf249e6b9" class="ne-image">

统计每一分钟，用户购买的商品的总数，代码如下：

```scala
// 用户id和购买数量 stream
val counts: DataStream[(Int, Int)] = ...
val tumblingCnts: DataStream[(Int, Int)] = \
counts.keyBy(0) // 用userId分组
      .timeWindow(Time.minutes(1)) // 窗口宽度为 1 分钟
      .sum(1) // 计算购买数量
```

## Sliding Windows
滑动窗口

+ 用于滚动进行聚合分析
+ 一个事件可以对应多个窗口
+ 实现滑动窗口：`timeWindow(窗口长度 size, 滚动时间 interval)`

例如：每隔 6 分钟统计一次过去一小时内所有商品的点击量，那么统计窗口彼此之间就是存在重叠的，即 1天可以分为 240 个窗口。

<img src="https://cdn.nlark.com/yuque/0/2025/png/54426354/1758288993429-df6cc421-1915-4750-a567-7d53e866c072.png" width="800" title="" crop="0,0,1,1" id="u35c9dd3a" class="ne-image">

可以看到 window 1 - 4 这四个窗口彼此之间都存在着时间相等的重叠部分。

以每隔 30 秒，统计最近 1 分钟用户购买的商品总数，代码如下

```scala
val slidingCnts: DataStream[(Int, Int)] = buyCnts
.keyBy(0) 
.timeWindow(Time.minutes(1), Time.seconds(30))
.sum(1)
```

## Session Windows
会话窗口：窗口由一段时间内不活跃的间隙分隔开。

+ 即如果在一段时间内没有数据到达，那么之前的窗口结束
+ 新的数据到达会开启新的窗口

<img src="https://cdn.nlark.com/yuque/0/2025/png/54426354/1758289240066-cde73c02-8163-4a4c-bfef-3d5aa69b3f0b.png" width="792" title="" crop="0,0,1,1" id="u2879d708" class="ne-image">

计算每个用户在活跃期间总共购买的商品数量，如果用户 30 秒没有活动则视为会话断开

```scala
// Stream of (userId, buyCnts)
val buyCnts: DataStream[(Int, Int)] = ...

val sessionCnts: DataStream[(Int, Int)] = vehicleCnts
.keyBy(0)
// session window based on a 30 seconds session gap interval 
.window(ProcessingTimeSessionWindows.withGap(Time.seconds(30)))
.sum(1)
```

## Global Windows
最后一个窗口是全局窗口， 全局窗口会将所有 key 相同的元素分配到同一个窗口中，其通常配合触发器（trigger）进行使用。

如果没有相应触发器，则计算将不会被执行。

<img src="https://cdn.nlark.com/yuque/0/2025/png/54426354/1758289309844-4e0b826f-234a-4c24-9477-041e7b8d1c2f.png" width="820" title="" crop="0,0,1,1" id="u4c9f18f6" class="ne-image">

这里继续以上面词频统计的案例为例，示例代码如下：

```java
// 当单词累计出现的次数每达到10次时，则触发计算，计算整个窗口内该单词出现的总数
window(GlobalWindows.create()).trigger(CountTrigger.of(10)).sum(1).print();
```

# Count Windows
Count Windows 用于以数量为维度来进行数据聚合

同样也分为滚动窗口和滑动窗口

实现方式也和时间窗口完全一致，只是调用的 API 不同，具体如下：

```java
// 滚动计数窗口，每1000次点击则计算一次
countWindow(1000)
// 滑动计数窗口，每10次点击发生后，则计算过去1000次点击的情况
countWindow(1000, 10)
```

```scala
// Stream of (userId, buyCnts)
val buyCnts: DataStream[(Int, Int)] = ...

val tumblingCnts: DataStream[(Int, Int)] = buyCnts
// key stream by sensorId
.keyBy(0)
// tumbling count window of 100 elements size
.countWindow(100)
// compute the buyCnt sum 
.sum(1)
```
