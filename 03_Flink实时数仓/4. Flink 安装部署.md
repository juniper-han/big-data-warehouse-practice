# Flink Local 安装部署
Local 模式是 Flink 提供的最简单部署模式，一般用来本地测试和演示使用。



下载 Apache Flink 1.10.0 for Scala 2.11 版本进行演示，该版本对应 Scala 2.11 版本。

将压缩包下载到本地，并且直接进行解压，使用 Flink 默认的端口配置，直接运行脚本启动：

`tar -zxvf flink-1.10.0-bin-scala_2.11.tgz`



运行脚本启动 Flink ：`./bin/start-cluster.sh `

访问本地的 8081 端口，可以看到 Flink 的后台管理界面，验证 Flink 是否成功启动

<img src="https://cdn.nlark.com/yuque/0/2026/png/54426354/1767600829878-7b59022a-6f62-4e05-b667-a0b60daed8a2.png" width="1920" title="" crop="0,0,1,1" id="ube9819d9" class="ne-image">



提交一个测试任务：`./bin/flink run examples/batch/WordCount.jar`

# Flink 集群安装部署
Flink支持多种集群安装部署方式

+ Standalone：独立环境
+ ON YARN：多用于生产环境，可以综合利用集群资源
+ Mesos、Kubernetes、AWS…

## Standalone 部署
### Standalone 部署操作步骤
架构图：  
<img src="https://cdn.nlark.com/yuque/0/2026/png/54426354/1767600063180-e70fc615-c019-4043-94ac-33f7440aac41.png" width="576" title="" crop="0,0,1,1" id="uf9ca9ba9" class="ne-image">

+ JDK1.8 及以上，并配置 JAVA_HOME 环境变量  
+ SSH 免密登录

集群规划：

+ Master：bigdata01
+ Slave：bigdata02，bigdata03

下载`flink-1.17.2-bin-scala_2.12.tgz`

1. 上传到 bigdata01 的 /data/soft 目录下，解压

```shell
tar -zxvf flink-1.17.2-bin-scala_2.12.tgz
```

2. 修改配置：`flink-1.17.2/conf`目录
    - 修改`flink-conf.yaml`文件

```yaml
     # 修改 jobmanager 为 bigdata01，原为localhost
     jobmanager.rpc.address:bigdata01
```

    - 修改`masters`文件

```yaml
     # 将localhost 改为 bigdata01
     bigdata01:8081
```

    - 修改`workers`文件

```yaml
     # 删除 localhost，新增bigdata02、bigdata03
     bigdata02
     bigdata03
```

3. 将 flink 的安装目录复制到其他两个节点

```shell
scp -rq flink-1.17.2 bigdata02:/data/soft/
scp -rq flink-1.17.2 bigdata03:/data/soft/
```

4. 启动 flink 集群，在 bigdata01 flink 安装目录下输入`bin/start-cluster.sh`
5. 验证
    - bigdata01：`jps`，出现`StandaloneSessionClusterEntrypoint`进程
    - bigdata02 和 bigdata03：`jps`，出现`TaskManagerRunner`进程
6. 访问 Flink Web 页面：`http://bigdata01:8081`
7. 停止机集群，在bigdata01 flink 安装目录下输入`bin/stop-cluster.sh`

### Standalone集群核心参数
| 参数 | 解释 |
| --- | --- |
| `jobmanager.memory.process.size` | 主节点可用内存大小 |
| `taskmanager.memory.process.size` | 从节点可用内存大小 |
| `taskmanager.numberOfTaskSlots` | 从节点可以启动的进程数量，建议设置为从节可用的cpu数量 |
| `parallelism.default` | Flink任务的默认并行度 |


### slot V.S. parallelism
+ slot 是静态的概念，是指 taskmanager 具有的并发执行能力  
+ parallelism 是动态的概念，是指程序运行时实际使用的并发能力  
+ 设置合适的 parallelism 能提高程序计算效率，太多了和太少了都不好

## Flink ON YARN
Flink ON YARN 模式：

+ 使用客户端的方式，直接向 Hadoop 集群提交任务
+ 不需要单独启动 Flink 进程
+ 依赖 Hadoop 2.4.1 及以上版本
+ 支持两种使用方式

Flink ON YARN 的优势

+ 提高大数据集群机器的利用率
+ 一套集群，可以执行 MR 任务，Spark 任务，Flink 任务等

### Flink ON YARN 的两种使用方式
<img src="https://cdn.nlark.com/yuque/0/2026/png/54426354/1767600141113-9288a32a-992d-47df-b8f5-3bc93f82fa1a.png" width="1623" title="" crop="0,0,1,1" id="u9bf31186" class="ne-image">

方式 1：

+ 在 YARN 中初始化 1 个 Flink 集群并常驻，需手工停止，提交的任务都提交到这个 Flink 集群中
+ 适合运行：规模小，短时间运行的作业

方式 2：

+ 每次提交任务都创建 1 个新的 Flink 集群，任务完成后，集群消失。任务间相互独立，方便管理
+ 适合运行：长时间运行的作业

### Flink ON YARN 的方式 1
+ 第 1 步：在 YARN 集群中，使用`yarn-session.sh`脚本，初始化 1 个长时间运行的 Flink 集群
+ 第 2 步：使用`flink run`命令向 Flink 集群提交任务

选择 bigdata02 作为 Flink 客户端所在机器  
第一步：安装 Flink 客户端：

+ 把 Flink 的安装包上传上去解压，不需要启动
+ 设置`HADOOP_CLASSPATH`环境变量，否则执行`yarn-session.sh`报错，找不到 Hadoop 的依赖

```shell
vi /etc/profile
# 新增
export HADDOOP_CLASSPATH=`{HADDOP_HOME}/bin/hadoop classpath`
```

+ 使用 Flink 安装目录下的`bin/yarn-session.sh`在 YARN 中创建一个长时间运行的 Flink 集群

```shell
yarn-session.sh -jm 1024m -tm 1024m -d
# -jm：指定主节点的内存
# -tm：指定从节点的内存
# -d：后台执行该进程
```

+ 启动后，日志信息会显示 Flink Web 界面的地址，以及该 Flink 集群在 YARN 中对应的applicationID
+ 在 YARN Web 界面可以看到该 Flink 集群，点击 Tracking UI 栏中的 ApplicationMaster 也可进入 Flink Web 界面

第二步：使用 Flink 安装目录下的`bin\flink run`向 Flink 集群提交任务，以 Flink 内置案例为例

```shell
bin/flink run ./examples/batch/WordCount.jar
```

+ 注意：使用`flink run`时，默认寻找`/tmp/.yarn-properties-root`文件，该文件中保存了 Flink 集群在 YARN 中的 applicationID
+ 查看 Flink Web，显示有一完成的任务，任务在执行时，会动态申请资源，任务执行完毕后，对应的资源会自动释放

第三步：`yarn application -kill`命令，停止集群

```shell
yarn application -kill 此处为 Flink 集群的 applicationID
```



`yarn-session.sh`的参数说明

+ `-help`：查看该命令的全部参数
+ `-D <arg>`：动态属性
+ `-d, --detached`：后台独立运行
+ `-j, --jar <arg>`：指定 Flink 任务的 JAR 包，此参数可省略
+ `-jm, --jobManagerMemory <arg>`：JobManager 的内存，MB
+ `-nm, --name`：在 YARN 上为该应用设置名字
+ `-q, --query`：显示 YARN 中可用的资源（内存，CPU核数）
+ `-qu, --queue <arg>`：指定 YARN 队列
+ `-s, --slots <arg>`：每个 TaskManager 使用的 slots 数量
+ `-tm, --taskManagerMemory <arg>`：每个 TaskManager 的内存，MB

### Flink ON YARN 的方式 2
方式 2：使用`flink run -m yarn-cluster`直接创建临时集群，并提交任务

```shell
bin/flink run -m yarn-cluster  -yjm 1024 -ytm 1024 ./examples/batch/WordCount.jar
```

`flink`命令可以使用`yarn-session.sh`中的命令，需要添加前缀`y`

+ `yjm`：JobManager 的内存，MB
+ `ytm`：TaskManager 的内存，MB

`flink run`的其他常用参数

+ `-c, --class <classname>`：动态指定 JAR 包入口类
+ `-m, --jobmanager <host:port>`：动态指定 JobManager（主节点）地址
+ `-p, --parallelism <parallelism>`：动态指定程序的并行度，可以覆盖配置文件中的默认值

## 提交 Flink 任务：以创建临时集群的方式
第一步：`pom.xml`中添加打包配置

```xml
<build>
    <plugins>
        <!-- java maven编译插件 -->
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.9.0</version>
            <configuration>
                <source>1.8</source>
                <target>1.8</target>
                <encoding>UTF-8</encoding>
            </configuration>
        </plugin>
        <!-- scala编译插件 -->
        <plugin>
            <groupId>net.alchim31.maven</groupId>
            <artifactId>scala-maven-plugin</artifactId>
            <version>3.3.1</version>
            <configuration>
                <scalaCompatVersion>2.12</scalaCompatVersion>
                <scalaVersion>2.12.11</scalaVersion>
            </configuration>
            <executions>
                <execution>
                    <id>compile-scala</id>
                    <phase>compile</phase>
                    <goals>
                        <goal>add-source</goal>
                        <goal>compile</goal>
                    </goals>
                </execution>
                <execution>
                    <id>test-compile-scala</id>
                    <phase>test-compile</phase>
                    <goals>
                        <goal>add-source</goal>
                        <goal>testCompile</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
        <!-- 打包插件 -->
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-assembly-plugin</artifactId>
            <version>2.6</version>
            <configuration>
                <descriptorRefs>
                    <descriptorRef>jar-with-dependencies</descriptorRef>
                </descriptorRefs>
                <archive>
                    <manifest>
                        <!-- 可以设置 Jar 包入口类，可选 -->
                        <mainClass></mainClass>
                    </manifest>
                </archive>
            </configuration>
            <executions>
                <execution>
                    <id>make-assembly</id>
                    <phase>package</phase>
                    <goals>
                        <goal>single</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>

```

注意：需要将 Flink 和 Hadoop 的相关依赖的`scope`属性设置为`provided`，这些依赖不需要打进JAR 包里面，集群中有。

第二步：生成 JAR 包：`mvn clean package -DskipTests`

第三步：将生成的 JAR 包上传到 Flink 客户端所在机器

第四步：提交 Flink 任务

```shell
bin/flink run -m yarn-cluster -c org.songjie.scala.SocketWindowWordCountScala  -yjm 1024 -ytm 1024 db_flink-1.0-SNAPSHOT-jar-with-dependencies.jar
```

**停止任务**

+ 不能通过`ctrl + c`的方式停止任务，因为 Flink 任务已经提交到 Hadoop 集群中了
+ 方式一：通过 YARN 停止任务

```shell
yarn application -kill application_1768962956138_0001
```

+ 方式二：通过 Flink 停止任务
    - 方式 1：在 Flink Web UI 中点击 `Cancel Job`
    - 方式 2：`bin/flink cancel -yid application_1768962956138_0001 7b99bfb261a92f84a89d87bcca3a3e23

这个 Flink 任务停止之后，对应 yarn-session（Flink集群）也就停止了

## 配置`historyserver`
目的：Flink 任务停止后，Flink Web 界面就无法查看，若要查看历史任务的执行信息，需启动`historyserver`进程

`historyserver`进程可以在任意一台机器上启动

选择在 Flink 客户端所在机器：bigdata02 启动

1. 修改 bigdata02 的`flink-conf.yaml`配置文件

```yaml
jobmanager.archive.fs.dir: hdfs://bigdata01:9000/completed-jobs/
historyserver.web.address: 192.168.136.102
historyserver.web.port: 8082
historyserver.archive.fs.dir: hdfs://bigdata01:9000/completed-jobs/
historyserver.archive.fs.refresh-interval: 10000
```

2. 启动`historyserver`

```shell
bin/historyserver.sh start
```

3. 验证：`jps`，出现`HistoryServer`进程
4. 注意：Hadoop 集群中的 `historyserver`进程也需要启动，在三台机器的 Hadoop 安装目录下，执行

```shell
bin/mapred --daemon start historyserver
```
