HBase 存储结构：MemStore → StoreFile → HFile

四个维度：存储层次、内部结构、数据流转、Compaction机制

# HBase 存储层次架构
## 整体存储结构图
```plain
┌─────────────────────────────────────────────────────────────────────────┐
│                              Table (表)                                  │
└─────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                              Region (分区)                               │
│                    按 RowKey 范围切分的数据单元                             │
└─────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                           Store (列族)                                   │
│              每个 Column Family 对应一个 Store                              │
└─────────────────────────────────────────────────────────────────────────┘
                                      │
                    ┌─────────────────┴─────────────────┐
                    │                                   │
                    ▼                                   ▼
        ┌───────────────────┐               ┌───────────────────┐
        │    MemStore       │               │    StoreFile      │
        │    (内存存储)      │               │    (磁盘存储)      │
        │                   │               │                   │
        │  • 最新写入数据    │               │  • Flush后的数据   │
        │  • 读操作优先查找  │               │  • HFile格式      │
        │  • 达到阈值Flush   │               │  • 存储在HDFS     │
        └───────────────────┘               └───────────────────┘
                                                      │
                                                      ▼
                                            ┌───────────────────┐
                                            │      HFile        │
                                            │  (底层文件格式)    │
                                            └───────────────────┘
```

## 存储层次关系
| 层次 | 说明 | 数量关系 |
| --- | --- | --- |
| **Table** | HBase 表 | 1 个 Table |
| **Region** | 表按 RowKey 切分 | 1 个 Table = N 个 Region |
| **Store** | 每个列族一个 Store | 1 个 Region = M 个 Store（M = 列族数） |
| **MemStore** | 内存写缓存 | 1 个 Store = 1 个 MemStore |
| **StoreFile** | 磁盘文件 | 1 个 Store = N 个 StoreFile |
| **HFile** | 底层存储格式 | 1 个 StoreFile = 1 个 HFile |

# MemStore 详解
## MemStore 核心概念
| 特性 | 说明 |
| --- | --- |
| 存储位置 | RegionServer 的 JVM堆内存 |
| 数据结构 | 跳表（SkipList）+ TreeMap |
| 排序方式 | 按 RowKey + Column Family + Column Qualifier + Timestamp 排序 |
| 数据特点 | 写入**有序**、支持多版本、内存中可查询 |
| 持久化 | 非持久化，**依赖 WAL 保证数据安全** |

## Flush 过程详解
Flush：MemStore 中的数据刷写到磁盘中

Flush 流程如下

1. 标记 MemStore 为不可写
2. 创建新的 MemStore（后续写入进入新 MemStore） 
3. 将旧 MemStore 数据写入临时文件
4. 临时文件重命名为 StoreFile（HFile 格式）
5. 更新 Region 元数据，记录新 StoreFile
6. **<font style="background-color:#E8F7CF;">清理已 Flush 的 WAL 日志</font>**

## Flush 触发条件
1. MemStore大小阈值：单个 MemStore 达到此值触发 Flush
    - 参数配置：`hbase.hregion.memstore.flush.size`
    - 默认值：128MB
2. Region 总 MemStore 阈值：总 Memstore 达到 RegionServer 堆内存40%
    - 参数配置：`hbase.regionserver.global.memstore.size`
    - 默认值：0.4
3. WAL 数量阈值：WAL 文件数量达到阈值
    - 参数配置：`hbase.regionserver.max.logs`
    - 默认值：32
4. 手动触发：通过 API 或命令行手动 Flush
5. Region 关闭：Region 关闭时强制 Flush

## 数仓开发关注点
1. MemStore 大小影响写入性能和 Flush 频率
2. MemStore 过大会增加 GC 压力，过小会增加 StoreFile 数量
3. 写多读少场景，可适当增大 MemStore
4. 读多写少场景，可适当减小 MemStore，增加 BlockCache
5. <font style="background-color:#E8F7CF;">Flush 是同步操作，频繁 Flush 会影响写入性能</font>

# StoreFile 详解
## StoreFile 核心概念
| 特性 | 说明 |
| --- | --- |
| **本质** | StoreFile 是 HBase的逻辑概念，物理上是 HFile |
| **存储位置** | HDFS 分布式文件系统 |
| **不可变** | StoreFile 一旦生成不可修改（**追加式写入**） |
| **多版本** | 一个 Store 可以有多个 StoreFile |
| **管理方式** | 由 RegionServer 管理，HMaster 维护元数据 |


## StoreFile 生命周期
1. 写入 MemStore
2. Flush
3. StoreFile 生成
4. Compaction 合并
5. 删除清理

## StoreFile 控制参数
| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `hbase.hstore.blockingStoreFiles` | 10 | StoreFile 数量达到此值阻塞写入 |
| `hbase.hregion.max.filesize` | 10GB | Region 大小达到此值触发分裂 |
| `hbase.hstore.compaction.threshold` | 3 | Minor Compaction 触发阈值 |


StoreFile过多有什么影响？	

+ 读性能下降、Compaction压力大、元数据膨胀

## StoreFile 与读性能关系
读请求处理顺序

1. BlockCache（缓存）：最快（命中缓存）
2. MemStore（内存）：快（内存读取）
3. StoreFile 1（磁盘）：中等（文件少时）
4. StoreFile N（磁盘）：慢（文件多时）

<font style="background-color:#E8F7CF;">⚠️</font><font style="background-color:#E8F7CF;"> StoreFile越多，读性能越差（需要扫描更多文件）</font>

## BlockCache 机制
| 缓存类型 | 说明 | 适用场景 |
| --- | --- | --- |
| **Data Block** | 存储 HFile 的 Data Block | 读多写少场景 |
| **Bloom Filter** | 存储 Bloom Filter 数据 | 快速判断 Key 是否存在 |
| **Index Block** | 存储索引信息 | 加速 Block 定位 |


BlockCache 层级结构

1. L1 Cache（堆内）：BlockCache 实现，受 JVM 堆大小限制，快速访问
2. L2 Cache（堆外）：BucketCache 实现，不受 GC 影响，容量更大

# 总结对比表
| 组件 | 存储位置 | 持久化 | 排序 | 主要作用 |
| --- | --- | --- | --- | --- |
| **MemStore** | JVM堆内存 | 否（依赖WAL） | 跳表有序 | 写缓存、读加速 |
| **StoreFile** | HDFS | 是 | HFile格式 | 持久化存储 |
| **HFile** | HDFS | 是 | Block组织 | 底层存储格式 |
| **BlockCache** | JVM堆/堆外 | 否 | - | 读缓存加速 |




1. 写入流程：Client → WAL → MemStore → Flush → HFile → HDFS 
2. 读取流程：BlockCache → MemStore → StoreFile(HFile)  
3. 数据组织：Table → Region → Store → MemStore/StoreFile 
4. Compaction：Minor（部分合并）+ Major（全量合并+清理） 
5. 性能关键：MemStore大小、BlockCache命中率、StoreFile数量

# 数仓开发场景与优化建议
## 实时数仓写入优化
| 场景 | 问题 | 优化方案 |
| --- | --- | --- |
| 高频写入 | Flush 频繁，StoreFile 过多 | 增大 MemStore 阈值，批量写入 |
| 写入延迟 | WAL 同步开销大 | 异步 WAL（牺牲部分安全性） |
| Region 热点 | 单 RegionServer 压力大 | RowKey 加盐散列，预分区 |
| 读性能差 | StoreFile 过多 | 调整 Compaction 参数，增加 BlockCache |


## 存储参数调优建议
```xml
<!-- 写多读少场景（实时数仓明细存储） -->
<property>
    <name>hbase.hregion.memstore.flush.size</name>
    <value>268435456</value>  <!-- 256MB，减少Flush频率 -->
</property>
<property>
    <name>hbase.regionserver.global.memstore.size</name>
    <value>0.5</value>  <!-- 50%堆内存给MemStore -->
</property>

<!-- 读多写少场景（用户画像查询） -->
<property>
    <name>hbase.regionserver.global.memstore.size</name>
    <value>0.3</value>  <!-- 30%给MemStore -->
</property>
<property>
    <name>hfile.block.cache.size</name>
    <value>0.4</value>  <!-- 40%给BlockCache -->
</property>
```

## 监控指标建议
| 指标 | 告警阈值 | 说明 |
| --- | --- | --- |
| MemStore 使用率 | >80% | 可能触发 Flush 阻塞 |
| StoreFile 数量 | >50/Store | 读性能下降 |
| Compaction 队列 | >10 | Compaction 滞后 |
| BlockCache 命中率 | <50% | 缓存配置不足 |
| RegionServer GC时间 | >1s | 内存配置不当 |
