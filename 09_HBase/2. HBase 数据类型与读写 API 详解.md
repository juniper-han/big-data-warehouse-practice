# HBase 数据模型与数据类型
## 核心数据模型
```plain
┌─────────────────────────────────────────────────────────────┐
│                        HBase 表                              │
├─────────────────────────────────────────────────────────────┤
│  RowKey    │  Column Family  │  Column Qualifier  │  Value  │
│            │                 │                    │         │
│  row001    │  cf1            │  name              │  张三   │
│  row001    │  cf1            │  age               │  25     │
│  row001    │  cf2            │  order_id          │  O001   │
│  row002    │  cf1            │  name              │  李四   │
└─────────────────────────────────────────────────────────────┘
```

## 数据类型特点
| 特性 | 说明 |
| --- | --- |
| **存储类型** | 所有数据都以**字节数组（**`**byte[]**`**）**的形式存储 |
| **无强类型** | HBase 本身不区分 `String/Int/Long` 等类型 |
| **类型转换** | 由**客户端**负责序列化和反序列化 |
| **版本控制** | 每个 Cell 可以存储多个版本（通过`Timestamp`区分） |


注意：HBase存储的数据类型是：

**<font style="background-color:#E8F7CF;">❗</font>****<font style="background-color:#E8F7CF;"> 所有数据都是</font>**`**<font style="background-color:#E8F7CF;">byte[]</font>**`**<font style="background-color:#E8F7CF;">，类型由客户端转换</font>**

## 常用类型转换工具类
工具类：`org.apache.hadoop.hbase.util.Bytes`

+ 序列化（转成 Byte 数组）：`Bytes.toBytes(var)`
+ 反序列化（转回对象）：`Bytes.toXXX(var)`

```java
// HBase 提供的 Bytes 工具类
import org.apache.hadoop.hbase.util.Bytes;

// 常见类型转换
byte[] rowKey = Bytes.toBytes("user_001");
byte[] name   = Bytes.toBytes("张三");
byte[] age    = Bytes.toBytes(25);
byte[] score  = Bytes.toBytes(3.14);
byte[] time   = Bytes.toBytes(System.currentTimeMillis());

// 反向转换
String nameStr = Bytes.toString(name);
int ageInt     = Bytes.toInt(age);
long timeLong  = Bytes.toLong(time);
```

# Java API 读写操作详解
## 环境依赖
`hbase-client`

```xml
<!-- Maven 依赖 -->
<dependency>
    <groupId>org.apache.hbase</groupId>
    <artifactId>hbase-client</artifactId>
    <version>2.4.9</version>
</dependency>
```

## 连接初始化
获取连接对象

```java
import org.apache.hadoop.hbase.*;
import org.apache.hadoop.hbase.client.*;
import org.apache.hadoop.conf.Configuration;

public class HBaseClient {
    
    private static Connection connection;
    
    // 初始化连接（单例模式，连接复用）
    public static Connection getConnection() throws IOException {
        if (connection == null || connection.isClosed()) {
            Configuration conf = HBaseConfiguration.create();
            conf.set("hbase.zookeeper.quorum", "node1,node2,node3");
            conf.set("hbase.zookeeper.property.clientPort", "2181");
            conf.set("zookeeper.znode.parent", "/hbase");
            connection = ConnectionFactory.createConnection(conf);
        }
        return connection;
    }
}
```

## 写操作 API
### ✅ 单条写入（`Put`）
使用`Put`对象和`Table`对象的`put`方法

步骤

1. 获取待写入的表对象：通过表名
2. 创建`put`对象：传入参数，序列化后的 Rowkey
3. 分别写入这条数据的不同列：有多少列，写几条`addColumn`语句，语句中传入列簇、列名、值，顺序为`(列簇, 列名, 列值)`
4. 向表对象提交`put`对象
5. 关闭表连接

```java
public static void putOneRow() throws IOException {
    Connection conn = HBaseClient.getConnection();
    Table table = conn.getTable(TableName.valueOf("user_info"));
    
    Put put = new Put(Bytes.toBytes("user_001"));
    put.addColumn(Bytes.toBytes("cf1"), Bytes.toBytes("name"), Bytes.toBytes("张三"));
    put.addColumn(Bytes.toBytes("cf1"), Bytes.toBytes("age"), Bytes.toBytes(25));
    put.addColumn(Bytes.toBytes("cf1"), Bytes.toBytes("city"), Bytes.toBytes("北京"));
    
    table.put(put);
    table.close();
}
```

### ✅ 批量写入（List）
1. 创建`List<Put>`对象，即列表中的元素是`Put`对象，将多个单条`put`组合成列表
2. 循环写入列表：实例化每个`put`对象，添加到列表中
3. 向表对象直接提交`List<Put>`对象，即批量提交
4. 关闭表连接

```java
public static void putBatch() throws IOException {
    Connection conn = HBaseClient.getConnection();
    Table table = conn.getTable(TableName.valueOf("user_info"));
    
    List<Put> puts = new ArrayList<>();
    for (int i = 0; i < 1000; i++) {
        Put put = new Put(Bytes.toBytes("user_" + String.format("%04d", i)));
        put.addColumn(Bytes.toBytes("cf1"), Bytes.toBytes("name"), Bytes.toBytes("用户" + i));
        put.addColumn(Bytes.toBytes("cf1"), Bytes.toBytes("age"), Bytes.toBytes(20 + i % 50));
        puts.add(put);
    }
    
    table.put(puts);  // 批量提交
    table.close();
}
```

### ✅ 带时间戳写入（多版本）
时间戳类型：`long`

时间戳写入位置：列名之后，列值之前，即`(列簇, 列名, 时间戳, 列值)`

```java
public static void putWithTimestamp() throws IOException {
    Connection conn = HBaseClient.getConnection();
    Table table = conn.getTable(TableName.valueOf("user_info"));
    
    Put put = new Put(Bytes.toBytes("user_001"));
    long ts1 = System.currentTimeMillis();
    long ts2 = ts1 - 1000;  // 1秒前
    
    put.addColumn(Bytes.toBytes("cf1"), Bytes.toBytes("name"), ts1, Bytes.toBytes("张三"));
    put.addColumn(Bytes.toBytes("cf1"), Bytes.toBytes("name"), ts2, Bytes.toBytes("张小三"));
    
    table.put(put);
    table.close();
}
```

### ✅ 条件写入（`checkAndPut`）
使用`table`对象的`checkAndPut`方法，传入 `rowkey`、`列簇`、`列名`的 Byte 数组，并传入判断条件，满段条件为`true`才写入

```java
public static void checkAndPut() throws IOException {
    Connection conn = HBaseClient.getConnection();
    Table table = conn.getTable(TableName.valueOf("user_info"));
    
    byte[] row = Bytes.toBytes("user_001");
    byte[] cf = Bytes.toBytes("cf1");
    byte[] col = Bytes.toBytes("age");
    
    // 只有当age=25时才更新为26
    boolean success = table.checkAndPut(
        row, cf, col, 
        Bytes.toBytes(25),  // 期望值
        new Put(row).addColumn(cf, col, Bytes.toBytes(26))
    );
    
    System.out.println("更新成功: " + success);
    table.close();
}
```

## 读操作 API
### ✅ 单条查询（`Get`）
使用`Get`对象和`Table`对象的`get`方法

1. 获取 HBase 连接，和`Table`对象
2. 实例化`Get`对象：初始化参数为待查询数据的`rowkey`
3. 也可在`Get`对象中指定要查询的列
4. 通过`Table`对象的`get`方法获取返回值，返回值类型为`Result`
5. 解析查询结果：通过`Result`对象的`getValue`方法，入参为列簇、列名的 Bytes 数组
6. 关闭`Table`对象

```java
public static void getOneRow() throws IOException {
    Connection conn = HBaseClient.getConnection();
    Table table = conn.getTable(TableName.valueOf("user_info"));
    
    Get get = new Get(Bytes.toBytes("user_001"));
    // 可选：指定查询的列
    // get.addColumn(Bytes.toBytes("cf1"), Bytes.toBytes("name"));
    
    Result result = table.get(get);
    
    if (!result.isEmpty()) {
        String name = Bytes.toString(result.getValue(Bytes.toBytes("cf1"), Bytes.toBytes("name")));
        int age = Bytes.toInt(result.getValue(Bytes.toBytes("cf1"), Bytes.toBytes("age")));
        System.out.println("name: " + name + ", age: " + age);
    }
    
    table.close();
}
```

### ✅ 批量查询（List）
类似批量写入

1. 使用`List<Get>`泛型，直接传入`Table`对象的`get`方法
2. 返回的查询结果类型是`Result[]`，也是`List`
3. 循环解析`Result[]`

```java
public static void getBatch() throws IOException {
    Connection conn = HBaseClient.getConnection();
    Table table = conn.getTable(TableName.valueOf("user_info"));
    
    List<Get> gets = new ArrayList<>();
    gets.add(new Get(Bytes.toBytes("user_001")));
    gets.add(new Get(Bytes.toBytes("user_002")));
    gets.add(new Get(Bytes.toBytes("user_003")));
    
    Result[] results = table.get(gets);
    for (Result result : results) {
        if (!result.isEmpty()) {
            System.out.println(Bytes.toString(result.getRow()));
        }
    }
    
    table.close();
}
```

### ✅ 范围扫描（Scan）
1. 使用`Scan`对象和`Table`对象的`getScanner`方法
2. `Scan`对象：传入 Rowkey 的起始值和终止值（不含）
3. 设置批次数量：限制从服务端拉取的数据量
4. 返回结果类型：`ResultScanner`，成员类型为`Result`，循环解析

```java
public static void scanRange() throws IOException {
    Connection conn = HBaseClient.getConnection();
    Table table = conn.getTable(TableName.valueOf("user_info"));
    
    Scan scan = new Scan();
    scan.setStartRow(Bytes.toBytes("user_0001"));
    scan.setStopRow(Bytes.toBytes("user_0100"));  // 不包含此值
    scan.setCaching(100);  // 每次从服务端拉取100条
    scan.setBatch(50);     // 每次返回50条
    
    try (ResultScanner scanner = table.getScanner(scan)) {
        for (Result result : scanner) {
            System.out.println(Bytes.toString(result.getRow()));
        }
    }
    
    table.close();
}
```

### ✅ 过滤器查询（Filter）
1. 使用`Scan`对象和过滤器对象
2. 创建过滤器对象，初始化传入各值和比较算子
3. 确认规则：没有满足该过滤器规则的过滤掉
4. 给`Scan`对象增加该过滤器
5. 后续同 Scan 范围扫描

```java
public static void scanWithFilter() throws IOException {
    Connection conn = HBaseClient.getConnection();
    Table table = conn.getTable(TableName.valueOf("user_info"));
    
    Scan scan = new Scan();
    
    // 单列值过滤器
    SingleColumnValueFilter filter = new SingleColumnValueFilter(
        Bytes.toBytes("cf1"),
        Bytes.toBytes("city"),
        CompareOperator.EQUAL,
        Bytes.toBytes("北京")
    );
    filter.setFilterIfMissing(true);  // 没有该列则过滤掉
    
    scan.setFilter(filter);
    
    try (ResultScanner scanner = table.getScanner(scan)) {
        for (Result result : scanner) {
            System.out.println(Bytes.toString(result.getRow()));
        }
    }
    
    table.close();
}
```

### ✅ 多版本查询
1. 通过`Get`对象的`setMaxVersions(n)`方法，获取最近的 n 个版本
2. 查询范围值类型为`Result`，但是需要通过`Result`对象的`getColumnCells`获取不同版本的结果，返回值类型为`Cell`

```java
public static void getMultipleVersions() throws IOException {
    Connection conn = HBaseClient.getConnection();
    Table table = conn.getTable(TableName.valueOf("user_info"));
    
    Get get = new Get(Bytes.toBytes("user_001"));
    get.setMaxVersions(5);  // 获取最近5个版本
    
    Result result = table.get(get);
    List<Cell> cells = result.getColumnCells(Bytes.toBytes("cf1"), Bytes.toBytes("name"));
    
    for (Cell cell : cells) {
        System.out.println("Timestamp: " + cell.getTimestamp() + 
                          ", Value: " + Bytes.toString(CellUtil.cloneValue(cell)));
    }
    
    table.close();
}
```

# Flink SQL 集成 HBase
## 依赖配置
`flink-connector-hbase`

```xml
<!-- Flink HBase Connector 依赖 -->
<dependency>
    <groupId>org.apache.flink</groupId>
    <artifactId>flink-connector-hbase-2.4</artifactId>
    <version>1.17.0</version>
</dependency>
```

## Flink SQL DDL 定义
+ 一个列簇为一个字段：
    - 字段名：列簇名
    - 字段类型：`ROW`类型，`ROW<col1 DataType, col2 DataType, Col3 DataType...>`
+ `WITH`参数
    - HBase 连接器，Zookeeper 信息
    - HBase 中的表名
    - 缓存的条数和 TTL

```plsql
-- 创建 HBase 表
CREATE TABLE hbase_user_info (
  rowkey STRING,
  cf1 ROW<name STRING, age INT, city STRING>,
  cf2 ROW<order_id STRING, order_time BIGINT>,
  PRIMARY KEY (rowkey) NOT ENFORCED
) WITH (
  'connector' = 'hbase-2.4',
  'table-name' = 'user_info',
  'zookeeper.quorum' = 'node1:2181,node2:2181,node3:2181',
  'zookeeper.znode.parent' = '/hbase',
  'lookup.cache.max-rows' = '10000',
  'lookup.cache.ttl' = '10min'
);
```

## Flink SQL 写入示例
写入时：把每个列簇涉及到的列值拼接成`ROW`

```plsql
-- 从 Kafka 读取数据写入 HBase
INSERT INTO hbase_user_info
SELECT 
    CONCAT('user_', user_id) as rowkey,
    ROW(name, age, city) as cf1,
    ROW(order_id, UNIX_TIMESTAMP(order_time) * 1000) as cf2
FROM kafka_order_source;
```

## Flink DataStream API 写入
```java
import org.apache.flink.configuration.Configuration;
import org.apache.flink.connector.hbase.sink.HBaseSink;
import org.apache.flink.connector.hbase.sink.HBaseMutationSinkBuilder;

// 创建 HBase Sink
HBaseSink<UserInfo> hbaseSink = new HBaseMutationSinkBuilder<UserInfo>()
    .setTableName("user_info")
    .setZookeeperQuorum("node1:2181,node2:2181,node3:2181")
    .setMutator((userInfo, context) -> {
        Put put = new Put(Bytes.toBytes(userInfo.getRowKey()));
        put.addColumn(Bytes.toBytes("cf1"), Bytes.toBytes("name"), Bytes.toBytes(userInfo.getName()));
        put.addColumn(Bytes.toBytes("cf1"), Bytes.toBytes("age"), Bytes.toBytes(userInfo.getAge()));
        return put;
    })
    .build();

// 写入数据
dataStream.sinkTo(hbaseSink);
```

## Flink 维表查询（Lookup Join）
```plsql
-- HBase 作为维表进行关联
SELECT 
    o.order_id,
    o.amount,
    h.cf1.name as user_name,
    h.cf1.city as user_city
FROM kafka_orders o
JOIN hbase_user_info FOR SYSTEM_TIME AS OF o.proc_time h
ON o.user_id = h.rowkey;
```

# 数仓开发中的最佳实践
## 批量写入优化
```java
// 使用 AutoFlush 控制
table.setAutoFlushTo(false);  // 关闭自动刷新
table.setWriteBufferSize(10 * 1024 * 1024);  // 10MB 缓冲区

// 批量提交
List<Put> puts = new ArrayList<>();
for (Data data : dataList) {
    Put put = buildPut(data);
    puts.add(put);
    if (puts.size() >= 1000) {
        table.put(puts);
        puts.clear();
    }
}
if (!puts.isEmpty()) {
    table.put(puts);
}
```

## 常见错误与规避
| 错误 | 原因 | 解决方案 |
| --- | --- | --- |
| Region 热点 | RowKey 顺序递增 | 加盐或哈希散列 |
| 写入慢 | 频繁小批量写入 | 使用批量写入+关闭AutoFlush |
| 查询慢 | Scan 未指定范围 | 设置StartRow/StopRow |
| 内存溢出 | 缓存过大 | 调整BlockCache和MemStore配置 |
| 数据不一致 | 未使用WAL | 重要数据确保WAL开启 |
